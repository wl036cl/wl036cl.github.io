HTML 编码规范

发表于 2018-07-03

前言

HTML 作为描述网页结构的超文本标记语言，在前端开发中一直占据基础地位。本文档的目标是使 HTML 代码风格保持一致，使代码更容易被理解和维护。

基本规范

HTML5 doctype

在每个 HTML 页面开头使用这个简单地 doctype 来启用标准模式，使其每个浏览器中尽可能一致的展现。

字符编码

为了确保内容快速呈现，如无特殊要求，统一使用 UTF-8 字符编码。

Language attribute

根据 HTML5 规范：

  • 鼓励网站作者在 html 元素上指定 lang 属性，来指出页面的语言。这样做会有助于语言合成工具来确定怎样发音，以及翻译工具决定使用的规则，等等。

推荐

浏览器内核模式

Internet Explorer 支持使用兼容性 标签来指定使用什么版本的 IE 来渲染页面。如果不是特殊需要，通常通过 edge mode 来通知 IE 使用最新的兼容模式。
双核浏览器支持 标签来指定使用什么浏览器内核来渲染页面。如果没有特殊需求，通常使用 webkit 来使浏览器更好的支持 HTML5，CSS3 。

引入 CSS 和 JavaScript

根据 HTML5 规范, 通常在引入 CSS 和 JavaScript 时不需要指明 type，因为 text/css 和 text/javascript 分别是他们的默认值。
引入 CSS 时必须指明 rel=”stylesheet”。

此外，引入 CSS 必须在 标签里引入。对于引入 Javascript，除了基础库等比较基础性的脚本文件，其他都在靠近 body 结束标签前面引入。

缩进使用两个空格

建议统一使用 2 个空格作为一个缩进层级。
由于过长代码不易阅读与维护，每行最好不超过120个字符。

标签名称和标签属性统一使用小写

viewport

viewport meta tag可以设置可视区域的宽度和初始缩放大小，避免在移动设备上出现页面展示不正常。

最常见的做法加入 <meta> 标签

更多的自适应方案可自行搜索。
另外，为了使 viewport 正常工作，在页面内容样式布局设计上也要做相应调整，如避免绝对定位等。关于 viewport 的更多介绍，可以参见 Safari Web Content Guide

属性规范

属性引号用双引号

Boolean 属性

Boolean 属性指不需要声明取值的属性。XHTML 需要每个属性声明取值，但是 HTML5 并不需要。

了解更多内容，参考 WhatWG section on boolean attributes:

一个元素中 Boolean 属性的存在表示取值 true，不存在则表示取值 false。

简而言之，尽量不要为 Boolean 属性添加取值。

自定义属性

建议自定义属性必须以 data- 为前缀，便于阅读。

属性顺序

HTML 属性应该按照特定的顺序以保证易读性

  • class
  • id, name
  • data-*
  • src, for, type, href, value
  • title, alt
  • role, aria-*

Classes 是为高可复用组件设计的，所以他们处在第一位。Ids 更加具体而且应该尽量少使用（例如, 页内书签），所以他们处在第二位。

标签规范

闭合规则

不要在自动闭合标签的结尾处使用斜线，如 <br>、<hr>、<input>、<meta>、<source>、<img>

闭合标签，不可省略

标签语义化

标签使用必须符合标签嵌套规则，比如 div 不得置于 p 中，tbody 必须置于 table 中

更多标签的使用，请参考 W3Cschool 中 HTML5 标签 。

命名规范

对于需要自定义属性，属性名命名统一使用 data- 前缀，具体参考 custom-data-attribute

  • class 命名和 CSS 编码规范中的保持一致
  • id 命名，当作为 JS 钩子使用 时，以 J_ 为前缀，后面接大驼峰命名，且这类选择器不能出现在 css 中，例如 J_ExampleIdForJs。其他使用用法，例如充当锚点，label 对应 input 的 id，都正常默认使用即可。
  • 元素 id 必须保证页面唯一。
  • 建议 id、name、class 命名时，在避免冲突并描述清楚的前提下尽量短。

多媒体退化

为 img 标签添加 alt 属性以声明替代文本，src 不能为空；
在多媒体标签内部提供指示浏览器不支持该标签的说明，如 object、audio、video

音频应尽可能覆盖到如下格式：

  • MP3
  • WAV
  • Ogg
    视频应尽可能覆盖到如下格式：
  • MP4
  • WebM
  • Ogg

注释规范

注释可以写在 <!-- 和 --> 中

由于 html 代码一般不会经过预处理，出于安全考虑，html 代码中不能出现任何关于业务相关敏感信息的注释。
